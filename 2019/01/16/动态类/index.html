0X0001 类的创建

首先开看如下API
Class objc_allocateClassPair(Class superclass, const char *name, size_t extraBytes)
中的内部实现机制，根据superclass 调用static Class alloc_class_for_subclass(Class supercls, size_t extraBytes)创建 cls 与meta两个类模板。

其中，alloc_class_for_subclass(Class supercls, size_t extraBytes) 在不考虑SuperClaa为Swift情况下，实现机制calloc(sizeof(objc_class) + extraBytes); ；另外，在类模板创建中, cls 与meta 同为结构体objc_class

0X0002 类的关联

objc_initializeClassPair_internal(superclass, name, cls, meta);
此函数完成以下职责：

  • 初始化cls 与 meta 类的基本信息
  • 建立superclasses 与 metaclasses之间的联系
  • 构建类B+树
  • 创建的类存储到 static NXHashTable *allocatedClasses 中
    注意：在类的构建期间不会 ivar layout

0X0002 类的注册

通过void objc_registerClassPair(Class cls)来完成注册:
1.标记cls & meta cls flags :

  • RW_CONSTRUCTED：class allocated and registered
  • RW_CONSTRUCTING:class allocated but not yet registered
  • RW_REALIZING:class has started realizing but not yet completed it

2.调用addNamedClass(cls, cls->data()->ro->name);，添加到NXMapTable *gdb_objc_realized_classes; 中，方便查找

补充

NXHashTable

注意：
在NXHashTable结构中

prototype的hash函数决定了index的获取规则分为通过指针获取还是字符串获取，NXHashTable中的hash函数的参数一般式指针即(Ptr)
count代表存储的HashBucket的数量
nbBucket代表bucket容量的大小一般为2^n-1
bucket指向存储HashBucket的数组的指针
在HashBucket结构中

count代表该HahsBucket结构存储元素的数量
当一个元素时，直接存储到void *one中
当多于一个元素时，会存储到二维数组 void **many 中 。

NXMapTable

注意：
在NXMapTable中

pototype的hash函数决定了index的获取规则分为通过指针获取还是字符串获取，NXMapTable中的hash函数的参数一般式字符串(Str)
count代表存储的MapPair的数量
nbBucketMinusOne代表bucket容量的大小一般为2^n-1
buckets指向存储MapPair的数组的指针
在MapPair中

key一般代表指向str的指针，所以该结构一般用于存储有名称的对象，如selctor，class，protocol等
value一般存储与key存储的字符串相关联的结构体.
